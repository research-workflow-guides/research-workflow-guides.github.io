Step 3

Project Template

Download one starter project, open it in VS Code, and use it to confirm the full toolchain works.

Download the starter archive

For this first rollout, use the existing starter repository as the project template source.

1. Open the starter repository

Use the repository page below and download the ZIP archive from GitHub.

2. Unzip the archive

Keep the folder intact so you can open the project structure in VS Code.

Starter repository: research-workflow-guides/seed-documents →

Open the project in VS Code

1. Open the extracted folder in VS Code

Open the starter folder as a project, not just a single file.

2. Use the template TeX file as your first target

In the current starter repository, open `seed-document/seed-document.tex` as the first working document.

Run the first build

1. Trigger the build from LaTeX Workshop

Use the editor UI rather than a manual terminal build for this first check.

2. Open the PDF preview

Confirm that the PDF opens and that the project compiles without blocking errors.

3. Treat this as the final setup check

If the template builds, your LaTeX distribution, editor, and extension are working together.

Self-check

Before you move on

Confirm that the starter file opens in VS Code, the build completes, and a PDF preview is available.
