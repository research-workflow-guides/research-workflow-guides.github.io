Step 1

LaTeX Installation

Install MiKTeX, enable package-on-demand behavior, and confirm the base toolchain is available.

Default distribution
MiKTeX
Goal
Reach a working LaTeX installation with `latexmk` available.

Why MiKTeX

For this site, MiKTeX is the default distribution because it is lighter to start with, beginner-friendly, and can install missing packages as you work.

This page is not a TeX Live comparison guide. The implementation path here assumes you will continue with MiKTeX.

Official MiKTeX downloads →

Install MiKTeX

1. Download the installer

Use the official MiKTeX download page and choose the installer for your platform.

2. Finish the installation

Complete the normal installer flow, then open MiKTeX Console once after installation.

3. Update packages

Run updates in MiKTeX Console before moving on, so the package database is current.

4. Keep package installation on demand

The working assumption for this guide is that MiKTeX can install missing packages when needed.

5. Confirm `latexmk` is available

If `latexmk` is not available yet, install it through MiKTeX Console before continuing.

Platform notes

Windows

Windows is the simplest path in this guide. If later workflows need Perl tools, install Strawberry Perl.

macOS

Use the official MiKTeX installer for macOS, then verify both MiKTeX and Perl from the terminal.

Linux

Treat Linux as a supported-with-caveat path. Follow MiKTeX documentation for your distribution and verify the toolchain carefully.

Perl requirement

`latexmk` and later `latexdiff`-style workflows depend on Perl-based tooling.

perl --version
latexmk -v

On Windows, install Strawberry Perl if Perl is missing. On macOS and Linux, verify that Perl is already available before you proceed.

Self-check

Before you move on

Confirm that MiKTeX Console opens, `latexmk -v` works, and `perl --version` works if your setup needs Perl-based tools.
